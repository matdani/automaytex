EN Introduction

Automaytex is a high-performance Maya plugin designed to bridge the gap between 3D geometry and Generative AI. It automates a complex "Bake-Diffuse-Reproject" workflow, allowing artists to texture assets using state-of-the-art Diffusion Models (SDXL, Flux, SD1.5) directly within the Maya viewport.

Core Concept

The engine bakes geometric data into a specialized 2x2 collage, processes this data via ControlNet, and reprojects the resulting AI-generated texture back onto the original mesh with UV integrity.

Why the Tetrahedron?

Initial versions used a 6-plane box projection. However, this created excessive redundancy and inefficient aspect ratios for Diffusion Models. The current Tetrahedral Strategy uses 4 optimized viewpoints in a 2x2 grid, maximizing VRAM efficiency and reducing reprojection artifacts.

CAT Documentació

Automaytex és un plugin per a Autodesk Maya 2026 que automatitza la creació de textures mitjançant models de difusió. L'objectiu és extreure la informació visual d'una geometria, transformar-la amb IA i tornar-la a projectar de forma coherent.

Flux de Treball

  • Pre-step: Emmagatzematge de les UVs originals per garantir un retargeting perfecte.
  • Pas 01: Extracció de dades geomètriques (Normals i Depth) mitjançant l'API d'Arnold.
  • Pas 02: Generació amb IA (SDXL/Flux) utilitzant quantitització dinàmica segons la VRAM local.

Technical Specifications

Feature Specification
Host Platform Autodesk Maya 2026
Render Engine Arnold API (Python Implementation)
Supported Models Flux, SDXL, Stable Diffusion 1.5
Geometry Capture 4-point Rectangular Tetrahedron
Optimization Local Quantization based on RAM/VRAM

AI Usage & Validation

In the development of Automaytex, AI tools were utilized strategically:

  • Code Generation: LLMs helped optimize complex Arnold API calls and the mathematical rotation matrices for the tetrahedral camera setup.
  • Manual Validation: Every AI-generated shader node and projection network was manually audited in the Maya Hypergraph. We tuned the ControlNet weight distribution by hand to ensure geometric edges remained crisp and free of "hallucinations."
